# 主机功耗监控工具 — 设计文档

**Date:** 2026-06-06
**Status:** Approved

---

## Overview

常驻后台的 Windows 桌面工具，通过软件传感器持续采集整机功耗，计算实际电费，并提供完整的桌面仪表盘。纯本地运行，资源占用低。

---

- **技术栈:** .NET 8 + WPF（Windows 原生 UI），单进程架构
- **进程模型:** 单进程，系统托盘常驻 + 后台采样线程 + UI 线程分离
- **传感器库:** LibreHardwareMonitor (NuGet)，直接嵌入，无需用户额外安装
- **数据库:** Microsoft.Data.Sqlite，本地单文件 `.db`

## Core Modules

### 1. Sensor Collection Service (传感器采集服务)

**职责:** 通过 LibreHardwareMonitor 定时扫描所有硬件传感器，返回结构化功耗数据。

**采集流程:**
```
采样定时器触发 (默认每5秒，1-60秒可调)
    ↓
LibreHardwareMonitor 扫描所有传感器芯片寄存器
    ↓
分类提取:
  ├─ CPU Package Power (W)      ← 直接传感器读数
  ├─ CPU Cores Power (W)        ← 直接传感器读数
  ├─ GPU Power (W)              ← 直接传感器读数
  ├─ 内存 (W)                   ← 按 DIMM 数 × 3-5W 估算
  ├─ 主板芯片组 (W)             ← 固定 15-30W 估算
  ├─ 硬盘 (W)                   ← HDD 6W / SSD 3W 估算
  ├─ 网卡 (W)                   ← 固定 3-5W
  ├─ 风扇 × N (W)              ← 按 RPM 比例估算，每个 1-3W
  └─ 电源转换损耗 (~10%)        ← 总功率 × 0.1 估算
    ↓
输出: List<ComponentPower>
```

**ComponentPower 结构:**
```
ComponentPower {
    Name: string        // "CPU Package", "GPU Core", "内存", ...
    Watts: double       // 功率值
    Source: enum         // Sensor | Estimated
}
```

**关键行为:**
- "扫描"是读取传感器芯片寄存器（I2C/SMBus 协议），不是磁盘扫描，对硬件零磨损
- 传感器读取失败时跳过本次采样（不写脏数据），下次采样继续
- 估算值标记 `Source: Estimated`，UI 上以 `○` 区别于传感器读数的 `●`

### 2. Power Estimation Engine (功率估算引擎)

**职责:** 对无传感器的组件提供固定估算值。

**估算策略（均为静态值，不随负载变化）:**
- 内存: 检测 DIMM 数量，每个 4W（DDR5 稍高按 5W）
- 主板芯片组: 20W（桌面平台典型值）
- 硬盘: HDD 每块 6W，SSD 每块 3W
- 网卡: 5W
- 风扇: 读取风扇数量 × 2W
- 电源转换损耗: (传感器合计 + 估算合计) × 10%

**计算总功率:**
```
总功率 = Σ(传感器读数) + Σ(估算值) + 电源转换损耗
```

### 3. Billing Calculator (电费计算引擎)

**计算逻辑:**
```
采样间隔耗电量 (kWh) = 总功率 (W) × 采样间隔 (秒) / 3,600,000

示例：总功率 200W，间隔 5 秒
  = 200 × 5 / 3,600,000
  = 0.000278 kWh

累计耗电量 = Σ(每次采样的耗电量)
电费 = 累计耗电量 × 单价 (¥/kWh)
```

**汇总维度:**
- 今日电费（当天 0:00 至今）
- 本周电费（周一至今）
- 本月电费（1日至今）
- 自定义时间段查询

**电价:** 用户通过设置界面配置固定单价（¥/kWh），默认 ¥0.6。

### 4. Sampling Scheduler (采样调度)

**职责:** 管理采样生命周期。

- 通过 `System.Threading.Timer` 触发，默认 5 秒间隔
- 用户可配置采样频率（1-60 秒）
- 支持开始/暂停/停止
- 暂停时不采样不写库，恢复后继续累计当天统计
- 托盘图标状态指示：运行中 🟢 / 已暂停 🟡

## Data Storage

### Database: SQLite (power.db)

**power_samples 表（采样快照）:**

| Field | Type | Description |
|-------|------|-------------|
| id | INTEGER PK | 自增主键 |
| timestamp | TEXT | ISO 8601 时间戳 |
| total_watts | REAL | 总功率 (W) |
| sensor_json | TEXT | 各组件明细 JSON（格式见下方） |
| price_per_kwh | REAL | 采样时的电价快照 |

**sensor_json 格式:**
```json
[
  {"name": "CPU Package", "watts": 65.2, "source": "sensor"},
  {"name": "GPU Core", "watts": 120.0, "source": "sensor"},
  {"name": "内存", "watts": 8.0, "source": "estimated"},
  {"name": "主板", "watts": 20.0, "source": "estimated"},
  {"name": "电源损耗", "watts": 21.3, "source": "estimated"}
]
```

**config 表（配置）:**

| Field | Type | Description |
|-------|------|-------------|
| key | TEXT PK | 配置键 |
| value | TEXT | 配置值 |
| updated_at | TEXT | 修改时间 |

**数据生命周期:**
- 所有数据保留本地
- 超过 90 天的详细采样数据自动聚合为小时级均值，释放存储空间
- 存储预估：5 秒采样间隔下，每天约 17,280 条，90 天约 1.56M 条，每条约 200 字节，合计约 300MB
- `price_per_kwh` 为快照字段：用户中途修改电价时，历史采样仍按采样时的电价计算，保证历史电费准确性

## Error Handling

- **传感器扫描失败:** 跳过本次采样，日志记录，连续 3 次失败后通知用户（托盘气泡提示）
- **数据库写入失败:** 重试 1 次，仍失败则记录日志并提示磁盘空间不足
- **LibreHardwareMonitor 初始化失败:** 启动时弹窗提示 "无法访问硬件传感器，请以管理员权限运行"
- **程序启动时无历史数据:** 正常显示空状态（"开始采样后将在此显示数据"）
- **磁盘空间不足:** 自动清理过期数据，仍不足则暂停采样

## Testing Strategy

- **单元测试:** 电费计算引擎（xUnit），验证 kWh 转换和电费计算精度
- **单元测试:** 功率估算引擎，验证估算值在合理范围内
- **集成测试:** 传感器采集 + 数据写入流程
- **手动测试:** 与实际用电数据对比（可选：使用智能插座校准）
